Introduction

HTML (HyperText Markup Language) structures content on the web. 

<!-- An HTML document contains elements (tags) that describe structure and meaning -->
<p>This is a paragraph.</p>
<strong>This text is important.</strong>

Create HTML file

Create a new file named index.html and add the minimal boilerplate. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My First Page</title>
  </head>
  <body>
    Hello, world!
  </body>
</html>
Open the file in a browser (double-click or use a local server). The doctype helps browsers use standards-compliant rendering.

Edit HTML file

Use any text editor (VS Code, Sublime, Vim, Notepad). Save with a .html extension and refresh the browser to see changes. 

<!-- Change the page title and body text, then hit refresh -->
<title>Edited Page</title>
<body>Now with updated content.</body>

Declarative language

HTML is declarative: you declare what elements are, not how to render them step-by-step. 

<article>
  <h1>A Declarative Document</h1>
  <p>Browsers decide visual details while following standards.</p>
</article>

Header element

Use <header> for introductory content of a page or section. It can include logos, headings, and nav. 

<header>
  <h1>Site Title</h1>
  <nav>
    <a href="#intro">Intro</a>
    <a href="#structure">Structure</a>
  </nav>
</header>

Paragraph element

Paragraphs wrap blocks of text. Browsers add spacing above/below by default. 

<p>Each paragraph communicates one idea clearly.</p>
<p>Use multiple paragraphs for readability.</p>

In line & nested elements

Inline elements flow within text (e.g., <em>, <a>), while block elements (e.g., <p>, <section>) start on a new line. Elements can be nested. 

<p>This is a <em>nested</em> <strong>inline</strong> example with a <a href="#">link</a>.</p>

Markup language

HTML is a markup language: tags annotate content to give it structure and meaning. 

<blockquote cite="https://example.com">
  <p>“Semantics matter for accessibility and SEO.”</p>
  <cite>— Someone on the Web</cite>
</blockquote>

Section element and indentation

Use <section> for standalone thematic groups. Proper indentation improves readability. 

<section>
  <h2>Features</h2>
  <p>Short description.</p>
  <section>
    <h3>Sub-feature</h3>
    <p>Nested section content.</p>
  </section>
</section>

Unordered list

Use <ul> with <li> for bullet lists where order doesn’t matter. 

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Ordered list

Use <ol> when sequence matters. We can change numbering with the start attribute. 

<ol start="3">
  <li>Plan</li>
  <li>Build</li>
  <li>Ship</li>
</ol>

Line breaks

Use <br> for a line break inside a block of text. Prefer paragraphs for separate thoughts. 

<p>Address Line 1<br>Address Line 2</p>

Comments

Comments don’t render in the page but help document your code. 

<!-- TODO: Replace placeholder image before launch -->
<img src="placeholder.png" alt="Placeholder">

Links and anchor

Use <a> for hyperlinks. Add target="_blank" + rel="noopener" for safe new-tab links. Anchors can jump to IDs. 

<a href="https://developer.mozilla.org/" target="_blank" rel="noopener">MDN Web Docs</a>
<a href="#intro">Back to Introduction (anchor)</a>

Images

Include descriptive alt text for accessibility and better loading behavior with width/height. 

<img src="cat.jpg" alt="A gray cat looking at the camera" width="320" height="200">
<picture>
  <source srcset="cat.avif" type="image/avif">
  <source srcset="cat.webp" type="image/webp">
  <img src="cat.jpg" alt="A gray cat" width="320" height="200">
</picture>

Audio and video

Use built-in media players with controls and captions for accessibility. 

<audio controls>
  <source src="sound.ogg" type="audio/ogg">
  <source src="sound.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

<video controls width="400" poster="thumb.jpg">
  <source src="clip.webm" type="video/webm">
  <source src="clip.mp4" type="video/mp4">
  <track kind="captions" src="captions.vtt" srclang="en" label="English">
  Sorry, your browser doesn't support embedded videos.
</video>

Tables

Tables are for tabular data. Use <thead>, <tbody>, and scope attributes for accessibility. 

<table>
  <caption>Quarterly Revenue</caption>
  <thead>
    <tr>
      <th scope="col">Quarter</th>
      <th scope="col">Amount</th>
    </tr>
  </thead>
  <tbody>
    <tr><th scope="row">Q1</th><td>$10k</td></tr>
    <tr><th scope="row">Q2</th><td>$14k</td></tr>
  </tbody>
</table>

Forms

Forms collect user input. Always associate <label> with inputs for accessibility. 

<form action="/subscribe" method="post">
  <fieldset>
    <legend>Newsletter</legend>
    <label for="email">Email</label>
    <input id="email" name="email" type="email" placeholder="you@example.com" required>
    <button type="submit">Subscribe</button>
  </fieldset>
</form>

Structure HTML page

A semantic layout improves readability, accessibility, and SEO. Here we have a common skeleton. 

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Semantic Structure</title>
</head>
<body>
  <header>...branding & nav...</header>
  <main>
    <article>...primary content...</article>
    <aside>...related info...</aside>
  </main>
  <footer>...copyright & links...</footer>
</body>
</html>
